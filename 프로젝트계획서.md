# simsimhae(심심해) 프로젝트 계획서

## 1. 프로젝트 개요

### 1.1 프로젝트 명
**simsimhae(심심해)** - 심심할 때 찾는 재미있는 컨텐츠 종합 웹사이트

### 1.2 프로젝트 목적
- 구글 애드센스를 통한 광고 수익 창출
- 다양한 엔터테인먼트 컨텐츠 제공으로 높은 사용자 체류 시간 확보
- 바이럴성 높은 컨텐츠로 자연스러운 트래픽 유입 유도

### 1.3 타겟 사용자
- 연령대: 20-40대
- 특성: 모바일 친화적, SNS 활동 활발, 간단한 재미 추구
- 이용 시나리오: 출퇴근 시간, 점심시간, 휴식 시간 등 틈새 시간 활용

### 1.4 주요 컨텐츠
1. **AI 사주풀이**: 생년월일시 기반 AI가 해석하는 사주팔자
2. **뭐먹지?**: 시간대별(아침/점심/저녁/간식/야식) 랜덤 음식 추천

## 3. 사이트 구조 및 페이지 계획

### 3.1 사이트맵
```
simsimhae/
├── 메인 페이지 (/)
├── AI 사주풀이 (/saju)
│   ├── 입력 페이지
│   ├── 결과 페이지
│   └── 공유 페이지
├── 뭐먹지? (/food)
│   ├── 선택 페이지 (시간대 선택)
│   ├── 결과 페이지
│   └── 히스토리
├── 회원 시스템 (/auth)
│   ├── 회원가입 (/auth/signup)
│   ├── 로그인 (/auth/login)
│   ├── 마이페이지 (/mypage)
│   └── 프로필 수정 (/mypage/edit)
├── 커뮤니티 (향후 확장)
└── 정보 페이지
    ├── 이용약관 (/terms)
    ├── 개인정보처리방침 (/privacy)
    └── 문의하기 (/contact)
```

### 3.2 페이지별 상세 구성

#### 메인 페이지
- 헤더: 로고, 네비게이션, 로그인/회원가입 버튼
- 히어로 섹션: 사이트 소개 및 주요 기능 강조
- 컨텐츠 카드: AI 사주풀이, 뭐먹지? 바로가기
- 최근 활동: 최근 사주풀이 결과, 인기 음식 추천
- 광고 영역: 상단 배너, 사이드바 광고
- 푸터: 저작권 정보, 약관 링크

#### AI 사주풀이 페이지
- 입력 폼: 생년월일시, 성별 입력
- 결과 화면: AI 해석 결과, 운세 차트
- 댓글 섹션
- 공유 버튼 (카카오톡, 페이스북, 트위터, 링크 복사)
- 광고 영역: 결과 중간, 하단

#### 뭐먹지? 페이지
- 시간대 선택: 아침/점심/저녁/간식/야식
- 랜덤 추천 버튼 (애니메이션 효과)
- 결과 화면: 음식 이미지, 설명, 관련 링크
- 히스토리: 최근 추천 받은 음식 목록
- 댓글 섹션
- 공유 기능
- 광고 영역

#### 마이페이지
- 프로필 정보
- 내가 본 사주풀이 기록
- 내가 추천받은 음식 기록
- 내 댓글 목록
- 설정 (알림, 개인정보 수정)

## 4. 각 컨텐츠 기능 상세 설계

### 4.1 AI 사주풀이 기능

#### 4.1.1 입력 정보
- 생년월일 (양력/음력 선택)
- 출생 시간 (시/분)
- 성별
- 이름 (선택사항)

#### 4.1.2 처리 프로세스
1. 사용자 입력 검증
2. 음력-양력 변환 (필요시)
3. 사주팔자 계산 (천간지지)
4. AI API 호출 (OpenAI GPT-4 또는 Claude)
   - Prompt 설계: 사주 정보 + 해석 요청
   - 응답 형식: 구조화된 JSON (전반적 운세, 재물운, 애정운, 건강운, 조언)
5. 결과 저장 (DB)
6. 결과 화면 렌더링

#### 4.1.3 결과 표시 항목
- 사주팔자 차트 (년/월/일/시 기둥)
- AI 해석 내용
  - 전반적 운세
  - 재물운
  - 애정운
  - 건강운
  - 올해의 조언
- 공유 이미지 자동 생성 (OG 이미지)

#### 4.1.4 추가 기능
- 회원 가입 시 무제한 조회
- 비회원: 일 3회 제한
- 과거 조회 기록 저장 (회원)
- 즐겨찾기 기능

### 4.2 뭐먹지? 기능

#### 4.2.1 카테고리 구성
- 아침: 한식, 양식, 간단식 (예: 토스트, 시리얼, 죽 등)
- 점심: 한식, 중식, 일식, 양식, 분식
- 저녁: 한식, 중식, 일식, 양식, 치킨, 피자
- 간식: 디저트, 과일, 스낵
- 야식: 치킨, 피자, 떡볶이, 라면, 족발/보쌈

#### 4.2.2 음식 데이터베이스 구조
```json
{
  "id": 1,
  "name": "김치찌개",
  "category": "한식",
  "mealTime": ["점심", "저녁"],
  "imageUrl": "/images/foods/kimchi-jjigae.jpg",
  "description": "한국의 대표적인 찌개 요리",
  "tags": ["매운맛", "국물요리", "따뜻한"],
  "relatedLinks": ["레시피 링크", "맛집 검색 링크"]
}
```

#### 4.2.3 추천 알고리즘
- 기본: 랜덤 선택 (가중치 적용 가능)
- 고급 (향후):
  - 날씨 연동 (날씨에 따른 추천)
  - 사용자 선호도 학습
  - 최근 추천 제외 필터
  - 계절별 추천

#### 4.2.4 UI/UX
- 룰렛 애니메이션 효과
- 재추천 버튼 (마음에 안들면 다시)
- 음식 상세 정보 모달
- 지도 연동 (근처 맛집 찾기)

## 5. 회원/댓글/SNS 공유 기능 설계

### 5.1 회원 시스템

#### 5.1.1 회원가입
- 이메일 + 비밀번호 방식
- 소셜 로그인 (향후): 구글, 카카오, 네이버
- 필수 정보: 이메일, 비밀번호, 닉네임
- 선택 정보: 생년월일, 성별
- 이메일 인증 (선택적)

#### 5.1.2 로그인
- 세션 기반 인증
- JWT 토큰 사용
- 자동 로그인 (쿠키)
- 비밀번호 찾기 (이메일 인증)

#### 5.1.3 회원 등급
- 일반 회원: 모든 기능 사용 가능
- 탈퇴 회원: 데이터 30일 보관 후 삭제

### 5.2 댓글 기능

#### 5.2.1 기능 요구사항
- 각 컨텐츠별 댓글 작성
- 대댓글 (1단계)
- 좋아요/싫어요
- 신고 기능
- 관리자 삭제 권한

#### 5.2.2 데이터 구조
```json
{
  "id": 1,
  "contentType": "saju" | "food",
  "contentId": 123,
  "userId": 456,
  "parentCommentId": null,
  "content": "댓글 내용",
  "likes": 5,
  "dislikes": 0,
  "isDeleted": false,
  "createdAt": "2026-01-05T10:00:00Z"
}
```

#### 5.2.3 스팸 방지
- 회원만 댓글 작성 가능
- 동일 내용 연속 작성 방지
- 욕설 필터링
- 신고 누적 시 자동 숨김

### 5.3 SNS 공유 기능

#### 5.3.1 지원 플랫폼
- 카카오톡
- 페이스북
- 트위터(X)
- 링크 복사

#### 5.3.2 공유 데이터 최적화
- Open Graph 메타 태그 설정
- 동적 OG 이미지 생성
  - 사주풀이: 결과 요약 이미지
  - 뭐먹지: 음식 이미지 + 추천 텍스트
- UTM 파라미터 추가 (유입 추적)

#### 5.3.3 공유 인센티브
- 공유 횟수 표시
- 공유 시 포인트 적립 (향후)
- 공유로 들어온 방문자 수 표시

## 6. 기술 스택 제안

### 6.1 프론트엔드
#### 추천 스택
- **프레임워크**: Next.js 14+ (React 기반)
  - 이유: SSR/SSG 지원으로 SEO 최적화, 빠른 초기 로딩
- **언어**: TypeScript
- **스타일링**: Tailwind CSS + shadcn/ui
- **상태관리**: Zustand 또는 React Query
- **폼 관리**: React Hook Form + Zod (검증)
- **애니메이션**: Framer Motion

#### 대안 스택
- **프레임워크**: Vue.js 3 + Nuxt.js 3
- **스타일링**: Vuetify 또는 PrimeVue

### 6.2 백엔드
#### 추천 스택
- **런타임**: Node.js
- **프레임워크**: Express.js 또는 Nest.js
  - Express: 가볍고 유연
  - Nest.js: 구조화되고 확장성 좋음 (추천)
- **언어**: TypeScript
- **인증**: Passport.js + JWT
- **API 문서**: Swagger/OpenAPI

#### 대안 스택
- **프레임워크**: Django (Python) - AI 연동 용이
- **프레임워크**: Spring Boot (Java) - 대규모 확장성

### 6.3 데이터베이스
#### 추천 구성
- **관계형 DB**: PostgreSQL
  - 회원 정보, 댓글, 사주 결과 저장
  - ACID 보장 필요한 데이터
- **캐싱**: Redis
  - 세션 관리
  - 자주 조회되는 데이터 캐싱
  - 일일 조회 제한 카운팅

#### 대안
- **MySQL**: 널리 사용되는 관계형 DB
- **MongoDB**: NoSQL, 유연한 스키마 (사주 결과 저장에 적합)

### 6.4 AI 연동
#### 사주풀이 AI
- **OpenAI GPT-4o**: 한국어 성능 우수, 창의적 해석
- **Anthropic Claude 3.5 Sonnet**: 긴 컨텍스트, 정확한 분석
- **네이버 HyperCLOVA X**: 한국 문화 이해도 높음

#### 구현 방식
- REST API 호출
- Prompt Engineering
  ```
  당신은 30년 경력의 사주 전문가입니다.
  다음 사주팔자를 분석하고, 친근하고 이해하기 쉬운 언어로 해석해주세요.

  [사주 정보]
  년주: 갑자(甲子)
  월주: 병인(丙寅)
  ...
  ```
- 응답 캐싱 (동일 사주 재요청 방지)

### 6.5 인프라 및 배포
#### 추천 구성
- **호스팅**: Vercel (프론트엔드) + AWS/GCP (백엔드)
  - Vercel: Next.js 최적화, 무료 티어
  - AWS EC2/RDS 또는 GCP Compute Engine
- **도메인**: Cloudflare (CDN + DNS + DDoS 방지)
- **파일 저장**: AWS S3 또는 Cloudinary (이미지)
- **CI/CD**: GitHub Actions

#### 대안
- **호스팅**: Netlify, Railway, Render
- **통합 솔루션**: AWS Amplify, Firebase

### 6.6 모니터링 및 분석
- **웹 분석**: Google Analytics 4
- **에러 추적**: Sentry
- **성능 모니터링**: Vercel Analytics 또는 New Relic
- **광고 플랫폼**: Google AdSense

## 7. 광고 연동 계획

### 7.1 Google AdSense 준비
#### 승인 요건
- 최소 콘텐츠: 20개 이상 페이지
- 트래픽: 일 100명 이상 권장
- 품질:
  - 독창적인 콘텐츠
  - 사용자 친화적 디자인
  - 개인정보 처리방침 명시
  - 이용약관 명시
- 도메인 등록 후 6개월 이상 (권장)

#### 승인 전략
1. 사이트 안정화 (1-2개월)
2. SEO 최적화
3. 꾸준한 콘텐츠 업데이트
4. 자연스러운 트래픽 확보
5. 신청 (트래픽 확보 후)

### 7.2 광고 배치 전략
#### 고수익 위치
1. **헤더 광고**: 728x90 리더보드 (데스크톱)
2. **사이드바 광고**: 300x600 하프 페이지 (데스크톱)
3. **콘텐츠 중간 광고**: 336x280 레타각형
4. **모바일 하단 고정 광고**: 320x100
5. **자동 광고**: AI 최적화 배치

#### 광고 배치 원칙
- 사용자 경험 최우선
- 콘텐츠와 광고 비율 70:30
- 광고 개수: 페이지당 3-4개 이하
- 로딩 속도 영향 최소화

### 7.3 수익 극대화 전략
- **A/B 테스팅**: 광고 위치 및 크기 실험
- **반응형 광고**: 화면 크기별 최적화
- **고가치 키워드**: 사주, 운세, 음식 관련 키워드
- **체류 시간 증가**: 양질의 콘텐츠로 PV 증가
- **다양한 광고 네트워크**:
  - Google AdSense (주)
  - Kakao AdFit (보조)
  - 네이버 애드포스트 (검토)

### 7.4 예상 수익 시뮬레이션
