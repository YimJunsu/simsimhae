# SimSimHae 프로젝트 개발 계획서

## 1. 프로젝트 개요

- **사이트 명**: SimSimHae (임시명)
- **목표**:  
  구글 광고 수익(AdSense)을 기반으로 한 종합 커뮤니티형 웹사이트 구축
- **컨셉**:  
  SNS 유행, 일상 콘텐츠, 재미 요소를 결합한 가볍고 자연스러운 분위기의 서비스

---

## 2. 주요 서비스 구성

### 2.1 메인 페이지

- 전체적으로 **자연스럽고 편안한 UI**
- 트렌디하지만 과하지 않은 디자인
- 주요 서비스 진입 버튼 제공
    - 음식 추천 서비스
    - AI 사주 풀이 서비스
- UI 및 공통 규칙은 프로젝트 루트의  
  **`개발규칙_및_협업계획서.md`** 문서를 기준으로 개발

---

### 2.2 음식 추천 서비스 (핵심 기능)

#### 2.2.1 서비스 개요
- 사용자의 상황에 맞는 음식을 **랜덤 추천**
- 재미 요소(모션, 애니메이션)를 통한 체류 시간 증가
- 유저 친화적 UI + 요즘 트렌드 컬러 및 효과 적용

---

#### 2.2.2 사용자 선택 단계 (페이지 진입 시)

1. **식사 시간 선택**
    - 아침 / 점심 / 저녁 / 간식 / 야식

2. **음식 종류 선택**
    - 한식 / 중식 / 일식 / 양식 / 기타 / 전체

3. **누구와 함께?**
    - 혼자 / 연인 / 가족 / 친구 / 기타

---

#### 2.2.3 추천 프로세스

- 모든 선택 완료 후:
    - **랜덤 추첨 모션 애니메이션 실행**
    - 로딩 중 음식 관련 재미있는 텍스트 또는 아이콘 노출
- 이후 음식 메뉴 결과 출력

---

#### 2.2.4 추천 결과 UI 구성

- 음식 이미지 또는 이모지
- 음식명
- 예상 칼로리 정보
- 간단한 설명(선택)
- **공유 기능**
    - 현재 추천 결과 URL 복사 버튼 제공

---

#### 2.2.5 음식 데이터 정책

- 데이터 우선순위:
    1. 공공데이터 포털의 음식 메뉴 데이터
    2. 음식 메뉴 API 또는 공개된 데이터 소스
    3. 위 데이터가 부족할 경우 AI 생성 데이터 활용

- 필수 조건:
    - 메뉴 종류는 **최대한 다양**
    - 모든 메뉴에 **칼로리 정보 포함**
    - 세부 메뉴는 각각 다른 메뉴로 취급
        - 예: 햄버거 ≠ 치즈버거 ≠ 불고기버거

---

### 2.3 AI 사주 풀이 서비스

- 현재 단계: **준비중 페이지**
- 페이지 접속 시:
    - “서비스 준비중입니다” 문구 노출
    - 추후 업데이트 안내 텍스트 포함
- 컨셉: 사주 정보 항목(ex. 양/음력, 생일, 이름, 성별, 태어난 시간 등)입력후 AI가 풀이해주는 사주

---

## 3. 프로젝트 구조 규칙

- 컴포넌트 단위 개발 필수
- 페이지 단위로 `pages` 디렉토리 분리
- 공통 UI는 공통 컴포넌트로 관리
- 전체 프로젝트 구조는 개발 시작 전 한 번 점검 후 진행

---

## 4. 향후 확장 고려사항

- SNS 공유 최적화(Open Graph)
- 광고 배치 고려한 레이아웃 설계
- 모바일 반응형 필수
- 사용자 체류 시간 증가 요소 지속 추가
